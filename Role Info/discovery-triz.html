?? Principles

My Code

1

Research before you design

Never guess what users want. Study competitors, analyze patterns, collect data. Design with evidence, not opinions.

2

Map the flow before the screen

A beautiful screen with broken navigation is still broken. Map the user journey, data flow, and system interactions first.

3

Design systems, not pages

Components, tokens, patterns  build a system that scales. Every new feature should feel like it belongs.

4

The right stack makes the right design possible

Tailwind enables rapid UI iteration. Vue + Nuxt handles SSR beautifully. Ionic + Laravel ships full-stack mobile apps. Unity + C# powers 2D/3D games, AR/VR, and simulations. Match the stack to the vision.

?? Mindset

How I Think

I approach every project with three questions: What are the best examples of this kind of experience? (Design Research). How does the user move through this system? (Flow Mapping). What technology makes this vision real and maintainable? (Stack Fit).

I speak designer and engineer fluently. I translate between them. I am the bridge between what looks good and what works.
